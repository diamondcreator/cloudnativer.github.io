CloudNative concept

Containerization, Scheduling, Microservices, elasticity, Automation

In general, "cloudnative" is a way to build and run applications that takes advantage of the cloud computing delivery model.

CNCF defines "cloudnative" more narrowly, which means using open source software stack for containerization, in which each part of the application is packaged in its own container and dynamically choreographed to actively schedule and manage each part to optimize resource utilization and microservice oriented applications, so as to improve the overall flexibility and maintainability of the application.
